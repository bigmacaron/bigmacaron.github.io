### git
#### 초기설정

```
git init
```
깃설정할 폴더 설정 (해당 디렉토리로 가야함)

```
git config user.name "USERNAME"
```
이름 설정 
```
git config user.email "USEREMAIL"
```

```
git commit -m "COMMITMESSAGE"
```
커밋하기 -> 커밋 메세지를 꼭 추가해줌
초기에 커밋하면 오류가 뜸 
오류가 뜨는 이유는 커밋할 파일을 지정안해 줬기 때문 ```git add```
그래서 아래처럼 파일을 지정해줌
```
git add FIFLENAME
```
파일을 추가

```
git status
```
```
On branch master
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
        modified:   app.py

Changes not staged for commit:
  (use "git add <file>.......
             .
             .
             .
	modified : FILENAME
```
깃 상태 확인(아래 내용)
- Changes to be committed: 
+ 커밋에 반영될 내용
- Untracked files :  
+ staging area에 추가된 내용(커밋할것이 올라와 있다라는뜻)  
변경 사항은 있지만 커밋에 반영되지 않았음
- modified :
+ 변경되었지만 ```git add``` 되지 않은 파일 이거나 ```git commit```  할 파일 
  
풀어서 조금더 설명을 하자면  
1.  파일 A와 B  가있음
2. 둘다 ```git add``` 해서 ```git commit``` 함
3. 둘다 파일수정 했음
4. A 만 ```git add``` 했음 
5. ```git status``` 입력
6. A는 커밋 준비 되었음
7. B는 커밋 준비 되지 않았음을 나타냄

```
git add .
```
현재 프로젝트 파일에서 변경 사항이 생긴 파일을 모두 추가 함

```
git reset FIFLENAME
```
명령어를 이용해```git add``` 한 파일을 제거
```
Unstaged changes after reset:
M        FIFLENAME
```
이라고 뜬다면 변경 사항이 있으니 확인 하라는 것
